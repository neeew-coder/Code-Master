CSS Accessibility

Accessibility ensures users of all abilities can navigate and understand content. CSS plays a vital role in color contrast, focus indicators, readability, and screen reader compatibility.

📘 What You’ll Learn

  • How to ensure sufficient color contrast for text and UI elements
  • How to style :focus states for keyboard navigation
  • How to use scalable units for readable typography
  • How to avoid layout traps and hidden content

🎨 Color Contrast

.text {
  background-color: #ffffff;
  color: #1e3a8a; /* Dark blue on white = high contrast */
}

Ensure a contrast ratio of at least **4.5:1** for normal text and **3:1** for large text. Use tools like WCAG Contrast Checker to validate.

🧭 Focus Styling

button:focus {
  outline: 2px solid #3b82f6;
  outline-offset: 2px;
}

Focus indicators help keyboard users see where they are. Never remove outline unless you're replacing it with a visible alternative.

📐 Readability Helpers

body {
  font-size: 1rem;
  line-height: 1.6;
  font-family: system-ui, sans-serif;
}
  • Avoid very light gray text on white
  • Use clear font sizes and spacing
  • Don’t rely on color alone to convey meaning

🔊 Screen Reader Support

  • Use aria-hidden="true" to hide decorative content
  • Use semantic tags: <nav>, <main>, <footer>
  • Don’t use display: none for essential content — it’s fully removed

🧠 Lesson Summary

  • Use :focus and :focus-visible to highlight active elements
  • Ensure text contrast meets WCAG AA (minimum 4.5:1 for body text)
  • Use relative units like em, rem, % for scalable text
  • Avoid hiding content with display: none unless it’s truly irrelevant

🧪 Try It Yourself

Create a file named css-accessibility-demo.html and link it to style.css. Then:

  • Style buttons and links with visible :focus outlines
  • Test color contrast using online tools like WebAIM or Contrast Checker
  • Use clamp() or rem units for responsive font sizing
  • Ensure all interactive elements are reachable via keyboard
