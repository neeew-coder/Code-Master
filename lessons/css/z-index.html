CSS z-index

The z-index property controls the stacking order of overlapping elements. Higher values appear in front — but only for elements with a position value other than static.

📘 What You’ll Learn

  • How z-index affects stacking order
  • How to layer elements using position and z-index
  • How higher values bring elements to the front
  • How to resolve overlap issues in UI design

📄 Syntax

.positioned {
  position: absolute;
  z-index: 10;
}

👁️ Layering Preview

z-index: 1 (behind)
z-index: 5 (middle)
z-index: 10 (front)

📌 Behavior Notes

  • z-index works only on positioned elements (relative, absolute, fixed, sticky)
  • The higher the value, the closer to the viewer
  • Negative values push elements behind their peers
  • Parents create new stacking contexts that limit z-index comparisons

📐 Sample Combo

.parent {
  position: relative;
  z-index: 1;
}

.child {
  position: absolute;
  z-index: 5;
}

🧠 Lesson Summary

  • z-index only works on positioned elements (relative, absolute, fixed, sticky)
  • Higher z-index values appear above lower ones
  • Default stacking is based on HTML order if no z-index is set
  • Use z-index to manage modals, dropdowns, tooltips, and overlays
