Common CSS Errors

CSS errors won’t usually break a page completely — but they can stop styles from working as intended. Understanding and spotting these issues is key to clean, predictable design.

🚫 Syntax Mistakes

  • Missing semicolon between properties
  • Unmatched curly braces
  • Invalid property names or values
/* ❌ Incorrect */
p {
  color: blue
  font-size: 18px;
}

/* ✅ Correct */
p {
  color: blue;
  font-size: 18px;
}

🎯 Selector Issues

  • Typos in class or ID names
  • Using a class selector (.title) but forgetting to add the class in HTML
  • Overly specific selectors that miss the actual target
/* Targeting class .title */
.title {
  font-size: 24px;
}

/* HTML must match */
<h1 class="title">Welcome</h1>

📏 Overridden Styles

  • Multiple conflicting rules — last one wins due to cascade
  • Inline styles override external and internal styles
  • Specificity beats location — #id > .class > tag

🛠️ Debugging Tips

  • Use browser DevTools (F12) to inspect elements and see applied styles
  • Check for typos in selector names
  • Verify external CSS files are linked correctly
  • Look for override issues — use DevTools "Computed" panel to trace style origin

🧠 Lesson Summary

  • CSS errors often result in styles being ignored, not broken pages
  • Syntax mistakes include missing semicolons, unmatched braces, and invalid properties
  • Selector issues stem from typos, missing classes, or overly specific targeting
  • Inline styles and specificity can override external rules unexpectedly
  • DevTools help trace style origins and debug overrides visually
