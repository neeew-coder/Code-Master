CSS Grid Layout

CSS Grid is a powerful two-dimensional layout system that lets you define rows and columns simultaneously. It enables precise, responsive design patterns — making full-page and component layouts more declarative and manageable.

📦 Grid Container Setup

.grid-layout {
  display: grid;
  grid-template-columns: 1fr 2fr;
  grid-template-rows: auto 200px;
  gap: 16px;
}
  • display: grid turns container into a grid parent
  • grid-template-columns and rows define the track sizes
  • gap adds spacing between items (rows & columns)

📐 Track Units

Unit Meaning Example
pxFixed pixels200px
%Relative to grid container50%
frFraction of available space1fr 2fr
autoSize based on contentauto
minmax()Set min and max boundsminmax(200px, 1fr)

🎨 Item Placement

.item {
  grid-column: 1 / span 2;
  grid-row: 2;
}
  • grid-column and grid-row: define placement via start/end or span
  • Use grid-area shorthand to place both
  • Default items auto-flow in row direction

📊 Named Grid Areas

.grid-layout {
  display: grid;
  grid-template-areas:
    "header header"
    "nav main"
    "footer footer";
  grid-template-columns: 2fr 1fr;
  gap: 16px;
}

.header { grid-area: header; }
.nav    { grid-area: nav; }
.main   { grid-area: main; }
.footer { grid-area: footer; }
  • Named areas improve layout readability and logic
  • Combine with semantic HTML elements for clarity
  • Use consistent naming across templates and components

📱 Responsive Strategy

.grid-responsive {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
  gap: 16px;
}
  • auto-fill or auto-fit fills space fluidly
  • minmax() keeps blocks readable and stable
  • Perfect for cards, galleries, flexible content zones

🧠 Grid Tips

  • Use gap instead of margin for cleaner alignment
  • Named areas + semantic HTML = easier maintenance
  • Inspect grid in browser dev tools → "Layout" panel
  • Don’t forget shorthand like grid and grid-area for brevity
