CSS Specificity

Specificity is the system CSS uses to decide which rule wins when multiple selectors target the same element. It calculates "weight" based on how specific a selector is.

ğŸ” Example Conflict

p {
  color: gray;          /* specificity 0,0,1 */
}

.text {
  color: blue;          /* specificity 0,1,0 */
}

#main p {
  color: red;           /* specificity 1,0,1 */
}

If a paragraph has #main as an ancestor and class="text", it will be red because the ID selector wins.

ğŸ§  Specificity Tips

  • More IDs = higher specificity
  • !important overrides specificity but should be avoided unless necessary
  • Specificity is cumulative â€” multiple classes beat a single type
  • Inline styles beat all selectors from stylesheets

ğŸ§  Lesson Summary

  • Inline styles have the highest specificity (score: 1000)
  • ID selectors score 100, class/attribute/pseudo-class selectors score 10
  • Element and pseudo-element selectors score 1
  • More specific rules override less specific ones â€” regardless of order
  • !important overrides specificity but should be used sparingly
