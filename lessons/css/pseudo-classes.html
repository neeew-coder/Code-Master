CSS Pseudo-Classes

Pseudo-classes allow you to style elements based on their state, position, or interaction. They enhance dynamic responsiveness without needing extra HTML or JS.

📘 What You’ll Learn

  • How to style links and buttons with :hover, :active, and :visited
  • How to target elements by position using :first-child, :last-child, and :nth-child()
  • How to style form inputs using :focus
  • How pseudo-classes improve accessibility and interactivity

🖱️ Interaction States

button:hover {
  background-color: steelblue;
}

input:focus {
  border-color: blue;
}

button:active {
  transform: scale(0.98);
}

🔍 Structural Positioning

li:first-child {
  font-weight: bold;
}

li:last-child {
  color: blue;
}

li:nth-child(odd) {
  background-color: #e0f2ff;
}
  • First item
  • Second item
  • Third item
  • Last item

📋 Form States

input:required {
  border-left: 4px solid steelblue;
}

input:valid {
  background-color: #f0fff4;
}

input:invalid {
  background-color: #fff0f0;
}

🧠 Pseudo-Class Tips

  • Use :hover and :focus together for better accessibility
  • :nth-child(odd/even) is great for table row striping
  • :not() helps exclude elements: div:not(.alert)
  • :root targets <html> for global CSS variables

🎓 Full Reference Snapshot

Pseudo-ClassMeaning

:requiredMandatory form field

🧠 Lesson Summary

  • :hover applies styles when the user points to an element
  • :active applies during click or tap
  • :visited styles links after they’ve been clicked
  • :focus applies when an input or button is selected
  • :nth-child() targets elements by index — useful for zebra striping
