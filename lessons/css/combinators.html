CSS Combinators

Combinators let you target elements based on their position or relationship in the HTML tree. They enhance selector precision and help you apply styles only where needed.

CSS Combinators Explained covers descendant, child, adjacent sibling, and general sibling combinators — with clear visuals and practical examples.

📘 What You’ll Learn

  • How to use the descendant combinator (div p)
  • How to target direct children with (div > p)
  • How to style adjacent siblings (h2 + p)
  • How to apply styles to all following siblings (h2 ~ p)

📘 Types of Combinators

CombinatorSymbolMeaning
DescendantspaceAny level nested inside
Child>Direct child only
Adjacent Sibling+Next sibling (one after)
General Sibling~Any next siblings

🔍 Examples and Output

/* Descendant */
section p {
  color: blue;
}

/* Child */
section > p {
  font-weight: bold;
}

/* Adjacent Sibling */
h2 + p {
  margin-top: 0;
}

/* General Sibling */
h2 ~ p {
  font-style: italic;
}

This nested <p> is not bold.

This direct child <p> is bold.

🧠 Combinator Tips

  • Use > when you only want to style direct children, not nested ones
  • + applies to one immediate sibling — great for spacing
  • ~ styles all following siblings — useful for themes or announcements
  • Combine with classes for scoped precision

🧠 Lesson Summary

  • div p targets all <p> inside a <div>, regardless of depth
  • div > p targets only direct child <p> elements
  • h2 + p targets the first <p> immediately after an <h2>
  • h2 ~ p targets all <p> siblings that follow an <h2>
