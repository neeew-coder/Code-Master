CSS !important

The !important flag forces a CSS rule to override all others — regardless of selector specificity or source order. It’s powerful, but often used as a last resort when the cascade isn’t working as expected.

📄 Syntax

.element {
  color: green !important;
}

The `color` will be red — even if another rule with higher specificity tries to assign a different value.

🧠 !important Tips

  • Use sparingly — it breaks the natural cascade
  • Can override inline styles (rare)
  • Avoid overuse — makes debugging harder
  • Useful for third-party overrides or accessibility fixes
  • If multiple `!important` rules apply, specificity still decides winner

🧪 Conflict Example

p {
  color: black;
}

.text {
  color: blue;
}

p.text {
  color: green !important;
}

🧠 Lesson Summary

  • !important makes a rule take precedence over others — even more specific ones
  • Inline styles with !important beat external styles with !important
  • Use !important only when necessary — e.g., overriding third-party styles
  • Refactor selectors and cascade before resorting to !important
