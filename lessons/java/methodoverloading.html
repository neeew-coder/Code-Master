🔁 Java Method Overloading

Method overloading allows multiple methods to share the same name but differ in the number or type of parameters. This makes your code more intuitive and flexible, especially when performing similar operations with different inputs.

📘 Why Overload Methods?

Overloading lets you define multiple versions of a method to handle different data types or argument counts. Java decides which version to call based on the arguments passed.

📘 Overloading Syntax

static void print(String text) { ... }
static void print(int number) { ... }
static void print(String text, int number) { ... }

🔍 Explanation

All three print methods have the same name but different parameter lists. Java automatically picks the correct one based on the arguments provided.

🧠 Analogy: Think of a smartphone camera app. You tap once to take a photo, hold to record video, or swipe to open filters. Same button, different actions — just like overloaded methods.

🧠 Lesson Summary

Method overloading allows you to define multiple versions of a method with the same name but different parameter lists. This enables flexible logic reuse while keeping method names intuitive. Overloading is resolved at compile time, and it’s a key feature for building APIs, utility classes, and user-friendly interfaces. It’s distinct from overriding, which involves inheritance and runtime polymorphism.

🧪 Try It Yourself

// OverloadDemo.java
public class OverloadDemo {
  public static void main(String[] args) {
    greet();
    greet("Mark");
    greet("Mark", 3);
  }

  static void greet() {
    System.out.println("Hello!");
  }

  static void greet(String name) {
    System.out.println("Hello, " + name + "!");
  }

  static void greet(String name, int times) {
    for (int i = 0; i < times; i++) {
      System.out.println("Hi, " + name + "!");
    }
  }
}
