🧬 Java Class Attributes

Attributes (also called fields or member variables) define the state of a class. They can be instance (unique to each object) or static (shared across all objects). Access modifiers like private, public, and protected control visibility.

📘 Types of Attributes
  • Instance Attributes: Each object has its own copy.
  • Static Attributes: Shared across all instances of the class.
  • Access Modifiers: Control visibility (private, public, protected).

🧠 Lesson Summary

Attributes in Java are variables declared inside a class to represent the state of an object. They can be primitive or reference types, and their visibility is controlled by access modifiers. Encapsulation encourages using private attributes with public getters and setters. Understanding how attributes work is essential for designing clean, maintainable object-oriented code.

🧪 Try It Yourself

// AttributeDemo.java
class Student {
  private String name;
  private int age;

  public Student(String name, int age) {
    this.name = name;
    this.age = age;
  }

  public String getName() {
    return name;
  }

  public void setAge(int age) {
    this.age = age;
  }

  public void display() {
    System.out.println("Name: " + name + ", Age: " + age);
  }
}

public class AttributeDemo {
  public static void main(String[] args) {
    Student s = new Student("user", 21);
    s.display();
    s.setAge(22);
    s.display();
  }
}
