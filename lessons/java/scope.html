🔍 Java Scope

Scope in Java defines where a variable can be accessed or modified. Understanding scope helps prevent bugs and makes your code more predictable and secure.

📘 What You’ll Learn

  • How scope defines where a variable can be accessed or modified
  • How local variables exist only within methods or blocks
  • How instance variables belong to objects and persist across method calls
  • How static variables belong to the class and share state across instances
  • How shadowing and naming conflicts affect scope resolution
📘 Types of Scope
  • Local Scope: Variables declared inside a method or block. Only accessible within that block.
  • Class Scope (Field): Variables declared inside a class but outside methods. Accessible by all methods in the class.
  • Block Scope: Variables declared inside loops, if-statements, etc. Exist only within that block.

🧠 Lesson Summary

Scope in Java determines the visibility and lifetime of variables. Local variables are confined to blocks or methods, instance variables belong to objects, and static variables belong to the class itself. Understanding scope helps prevent naming conflicts, manage memory, and write clean, predictable code. It’s foundational to debugging, encapsulation, and modular design.

🧪 Try It Yourself

// ScopeDemo.java
public class ScopeDemo {
  static int staticCount = 0; // Class scope

  public static void main(String[] args) {
    int localCount = 5; // Method scope
    for (int i = 0; i < localCount; i++) {
      int loopVar = i * 2; // Block scope
      System.out.println("LoopVar: " + loopVar);
    }
  }
}
