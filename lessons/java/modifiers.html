🛡️ Java Modifiers

Modifiers in Java control how classes, methods, and variables behave. They define visibility, inheritance rules, and usage restrictions. Java has two main types: access modifiers and non-access modifiers.

📘 What You’ll Learn

  • How access modifiers control visibility: public, private, protected, and package-private
  • How non-access modifiers affect behavior: static, final, abstract, synchronized
  • How modifiers apply to classes, methods, and variables
  • How to combine modifiers for specific design goals (e.g., private static final)
  • How modifiers support encapsulation, inheritance, and immutability
📘 Types of Modifiers
  • Access Modifiers: public, private, protected, and default (no keyword).
  • Non-Access Modifiers: static, final, abstract, synchronized, etc.

🧠 Lesson Summary

Modifiers in Java define how classes, methods, and variables behave and who can access them. Access modifiers control visibility across packages and subclasses, while non-access modifiers define characteristics like immutability (final), shared state (static), or abstraction (abstract). Understanding modifiers is key to writing secure, maintainable, and well-structured Java code.

🧪 Try It Yourself

// ModifierDemo.java
public class ModifierDemo {
  private static final double PI = 3.14159;

  public static void main(String[] args) {
    System.out.println("PI: " + PI);
  }

  protected void greet() {
    System.out.println("Hello from a protected method!");
  }
}
