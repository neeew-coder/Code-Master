🧱 Java Object-Oriented Programming (OOP)

OOP is a programming paradigm based on the concept of "objects" — containers that hold data and behavior. Java is built around OOP, making it easier to model real-world systems.

📘 What You’ll Learn

  • How Java uses classes and objects to model real-world entities
  • How encapsulation protects data using access modifiers
  • How inheritance enables code reuse and hierarchy
  • How abstraction hides complexity and exposes essential behavior
  • How polymorphism allows flexible method behavior across types
📘 Four Pillars of OOP
  • Encapsulation: Bundling data and methods together, hiding internal details.
  • Inheritance: Reusing code by creating new classes from existing ones.
  • Polymorphism: Using one interface to represent different underlying forms.
  • Abstraction: Hiding complex implementation and showing only essential features.

🧠 Lesson Summary

Object-Oriented Programming (OOP) in Java organizes code around objects rather than actions. Classes define blueprints, and objects are instances with state and behavior. Encapsulation secures data, inheritance builds relationships, abstraction simplifies interfaces, and polymorphism enables dynamic behavior. Mastering OOP is key to building scalable, maintainable, and intuitive Java applications.

🧪 Try It Yourself

// OOPDemo.java
class Animal {
  void speak() {
    System.out.println("Animal speaks");
  }
}

class Dog extends Animal {
  void speak() {
    System.out.println("Dog barks");
  }
}

public class OOPDemo {
  public static void main(String[] args) {
    Animal a = new Dog(); // Polymorphism
    a.speak(); // Output: Dog barks
  }
}
