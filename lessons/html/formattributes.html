<form> Element Attributes

Attributes on the <form> element define how user input is submitted, processed, displayed, and validated. These attributes affect HTTP behavior, browser experience, and form accessibility.

📤 action

Sets the server endpoint that processes the form data when submitted.

<form action="/action_page.php">

💡 Tip: If action is omitted, the form submits to the same page.

🎯 target

Specifies where to display the server response after form submission.

ValueDescription
_blankOpen in a new tab/window
_selfOpen in same tab (default)
_parentOpen in parent frame
_topOpen in full browser window
framenameOpen in specified iframe
 
<form action="/action_page.php" target="_blank">

📡 method

Specifies how form data is submitted to the server.

  • GET – Appends data to the URL (visible, bookmarkable)
  • POST – Sends data in the request body (secure, size-friendly)
<form action="/action_page.php" method="post">

🔐 Always use POST for sensitive data.

✏️ autocomplete

Enables browser to autofill inputs based on prior entries. 

<form action="/action_page.php" autocomplete="on">

Use off to disable autofill.

🚫 novalidate

Prevents browser-based form validation. Useful when handling validation server-side or with custom JavaScript. 

<form action="/action_page.php" novalidate>
