HTML Headings

In this tutorial, you'll learn how to use HTML heading tags to organize content, improve readability, and structure your webpage for both users and search engines.

  • Why <h1> should be unique per page
  • How headings improve accessibility and SEO
  • How to nest headings logically within sections and articles

🔠 HTML Heading Tags

HTML headings are defined with the <h1> to <h6> tags.

<h1> defines the most important heading. <h6> defines the least important. 

<h1>Heading 1</h1>
<h2>Heading 2</h2>
<h3>Heading 3</h3>
<h4>Heading 4</h4>
<h5>Heading 5</h5>
<h6>Heading 6</h6>
  
<!DOCTYPE html>
<html>
  <body>
    <h1>Heading 1</h1>
    <h2>Heading 2</h2>
    <h3>Heading 3</h3>
    <h4>Heading 4</h4>
    <h5>Heading 5</h5>
    <h6>Heading 6</h6>
  </body>
</html>

📐 Default Styling

Browsers automatically add margin before and after headings. Each heading tag also has a default font size.

🔍 Why Headings Matter

  • Search engines use headings to index page structure
  • Users skim pages by headings — use them to guide reading
  • Use <h1> for main headings, followed by <h2>, <h3>, etc.
  • Don’t use headings just to make text big or bold — use them semantically

🎨 Custom Font Size

You can override the default size using the style attribute and CSS font-size: 

<h1 style="font-size:60px;">Heading 1</h1>
  
<!DOCTYPE html>
<html>
  <body>
    <h1 style="font-size:60px;">Heading 1</h1>
  </body>
</html>

🧠 Lesson Summary

  • <h1> is the main title — use only once per page
  • <h2> to <h6> define subsections
  • Headings should follow a logical nesting order
  • Screen readers use headings to navigate content
