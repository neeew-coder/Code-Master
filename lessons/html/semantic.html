Semantic Elements in HTML

Semantic elements clearly define their purpose in the HTML structure — helpful for developers, browsers, and accessibility tools. Instead of just saying “this is a container,” they say “this is a header” or “this is an article.”

📌 Semantic vs Non-Semantic

  • Non-semantic: <div>, <span> — no descriptive meaning
  • Semantic: <header>, <article>, <section> — clearly describes its role

📖 Common Semantic Tags

  • <header>
  • <footer>
  • <main>
  • <section>
  • <article>
  • <nav>
  • <aside>
  • <figure>
  • <figcaption>
  • <details>
  • <summary>
  • <mark>
  • <time>

🧩 Visual Structure Example

<header> Site Title + Logo
<nav> Navigation Links
<main>
<article> Blog Post
<section> News Block
<aside> Sidebar Widgets
<footer> Contact Info

🧠 Sample Code Snippet

<article>
  <header>
    <h1>What Does WWF Do?</h1>
    <p>WWF's mission:</p>
  </header>
  <p>
    WWF's mission is to stop the degradation of our planet's natural environment,
    and build a future in which humans live in harmony with nature.
  </p>
  <footer>
    <p>Author: Hege Refsnes</p>
  </footer>
</article>

📚 Quick Summary

Element Purpose
<header>Introductory content
<nav>Navigation links
<main>Main content
<article>Self-contained block
<section>Grouped themed content
<aside>Side or supplementary info
<footer>Document or block footer
