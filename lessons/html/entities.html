HTML Entities & Special Characters

HTML entities let you display characters that have reserved meaning, don't exist on your keyboard, or serve formatting purposes. They’re crucial for writing clean, readable HTML — especially in multilingual, math-heavy, or code-oriented projects.

📎 Why We Need Entities

  • Prevent browser misreading: Using &lt; instead of < avoids breaking HTML
  • Display special symbols: ©, ™, €, ₹ using &copy;, &trade;, etc.
  • Control whitespace: &nbsp; and &thinsp; allow precise layout control
  • Enable international characters: á, ñ, ç, Ø with accent or Unicode entities

🔣 Entity Types

  • Named: &lt;, &euro; (easier to remember)
  • Numeric: &#60;, &#8364; (universal across encodings)

🪙 Currency & Trademark Symbols

<p>&cent; &pound; &yen; &euro; &copy; &reg; &trade;</p>

Result: ¢ £ ¥ € © ® ™

📐 Spacing Control

Browsers collapse normal spaces — use entities for precision:

X&nbsp;Y&thinsp;Z

Result: X Y Z (wide, thin, normal spacing)

⚠️ Common Pitfalls

  • < must be escaped to &lt; when inside plain text
  • & must be escaped unless starting a valid entity like &amp;
  • Accented characters may not render without correct encoding (use <meta charset="UTF-8">)
