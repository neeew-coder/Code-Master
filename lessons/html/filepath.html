HTML File Paths

A file path describes where a file is located within your website’s folder structure. File paths are used to link images, stylesheets, scripts, or other pages.

📂 Common Relative Path Examples

<img src="picture.jpg">


<img src="images/picture.jpg">


<img src="/images/picture.jpg">


<img src="../picture.jpg">

🌐 Absolute File Path

An absolute path uses the complete URL to reference a file from anywhere on the internet: 

<img src="https://www.w3schools.com/images/picture.jpg" alt="Mountain">

Useful when linking to images or files hosted externally (CDN, third-party sites).

🧭 Relative File Paths

Relative paths reference files within your own project, based on the location of the HTML file:

  • images/picture.jpg → Inside a folder in the current directory
  • /images/picture.jpg → Inside the root-level images folder
  • ../images/picture.jpg → Inside a folder one level above

✅ Best Practice

Use relative paths whenever possible. This keeps your project portable and compatible across:

  • Local environments (e.g. localhost)
  • Public web hosting (e.g. yourdomain.com)
  • Future deployments or domain changes
