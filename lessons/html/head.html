HTML <head> Element

The <head> element is a container for metadata and non-visual resources. It sits between the opening <html> and <body> tags. Its contents define how the page is processed by browsers and search engines.

üìå What Can Go Inside <head>?

  • <title> ‚Äì Required document title
  • <style> ‚Äì Internal CSS styles
  • <meta> ‚Äì Charset, SEO, author, viewport
  • <link> ‚Äì External resources (like CSS)
  • <script> ‚Äì JavaScript references or inline scripts
  • <base> ‚Äì Sets a default URL or target for relative paths

üéØ Sample Head Section

<head>
  <title>A Meaningful Page Title</title>
  <meta charset="UTF-8">
  <meta name="description" content="Free Web tutorials">
  <meta name="keywords" content="HTML, CSS, JavaScript">
  <meta name="author" content="John Doe">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <link rel="stylesheet" href="mystyle.css">
  <style>
    body {background-color: powderblue;}
    h1 {color: red;}
    p {color: blue;}
  </style>
  <script>
    function myFunction() {
      document.getElementById("demo").innerHTML = "Hello JavaScript!";
    }
  </script>
  <base href="https://www.codemaster.ph/" target="_blank">
</head>

üåê What Does Each Element Do?

ElementPurpose
<title>Defines the page title (SEO, tab label)
<style>Internal CSS rules
<meta>Charset, SEO metadata, responsiveness
<link>Loads external stylesheets or resources
<script>Adds client-side JavaScript
<base>Defines default URL or target for relative links

üì± What Is the Viewport Meta Tag?

The <meta name="viewport"> tag improves mobile responsiveness. It tells the browser to scale the page according to the device screen: 

<meta name="viewport" content="width=device-width, initial-scale=1.0">

This helps your layout adapt across desktops, tablets, and phones.
