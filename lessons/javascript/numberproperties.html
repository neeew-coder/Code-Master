JavaScript Number Properties

The Number object provides static properties that describe numeric limits and special values. These are useful for validation, boundary checks, and understanding JavaScript's numeric behavior.

📄 Common Properties

Number.MAX_VALUE;           // Largest possible number
Number.MIN_VALUE;           // Smallest positive number
Number.NaN;                 // Not-a-Number value
Number.POSITIVE_INFINITY;   // Infinity
Number.NEGATIVE_INFINITY;   // -Infinity
Number.MAX_SAFE_INTEGER;    // 2^53 - 1
Number.MIN_SAFE_INTEGER;    // -(2^53 - 1)

These properties are read-only and accessed directly from the Number object. They help you handle edge cases and avoid precision errors.

🧠 Notes

  • Number.MAX_VALUE ≈ 1.7976931348623157e+308
  • Number.MIN_VALUE ≈ 5e-324 (smallest positive number)
  • Number.NaN is returned from invalid math like 0 / 0
  • Number.MAX_SAFE_INTEGER is 2^53 - 1
  • Use these properties to validate and guard against overflow
