JavaScript Operator Precedence

Operator precedence determines the order in which operations are evaluated in expressions. Higher precedence operators are evaluated before lower ones.

🧠 Lesson Summary

Operator precedence in JavaScript defines the order in which parts of an expression are evaluated. For example, multiplication has higher precedence than addition, so 2 + 3 * 4 evaluates as 2 + (3 * 4). Associativity determines direction — left-to-right or right-to-left — for operators with the same precedence. Parentheses can override these rules to force custom evaluation. Understanding precedence helps avoid bugs and write predictable logic.

🧪 Try It Yourself

  console.log(2 + 3 * 4);        // → 14 (multiplication first)
  console.log((2 + 3) * 4);      // → 20 (parentheses override)

  console.log(10 - 2 + 1);       // → 9 (left-to-right associativity)
  console.log(2 ** 3 ** 2);      // → 512 (right-to-left associativity)

  console.log(true && false || true); // → true (&& before ||)
