JavaScript Data Types

JavaScript variables can hold different types of data. These types determine how values behave, how they’re stored, and how they interact with other values. JavaScript is dynamically typed, meaning a variable’s type can change during execution.

📄 Primitive vs Reference Types

  • Primitive Types: immutable, stored by value
    • String — text ("Hello")
    • Number — integers or decimals (42, 3.14)
    • Boolean — true or false
    • Undefined — declared but not assigned
    • Null — explicitly empty
    • Symbol — unique identifiers (advanced)
    • BigInt — large integers beyond Number.MAX_SAFE_INTEGER
  • Reference Types: mutable, stored by reference
    • Object — key-value pairs ({ name: "User" })
    • Array — ordered list ([1, 2, 3])
    • Function — executable code blocks

📦 Syntax Example

let name = "User";               // String
let age = 25;                    // Number
let isStudent = true;           // Boolean
let score;                      // Undefined
let empty = null;               // Null
let user = { name: "User" };    // Object
let grades = [90, 85, 88];      // Array
let greet = function() { return "Hi"; }; // Function
let big = 12345678901234567890n; // BigInt

Use typeof to inspect a variable’s type. Note that typeof null returns "object" — this is a known quirk in JavaScript.

🧠 Notes

  • typeof null returns "object" — a legacy bug
  • typeof [] also returns "object" — use Array.isArray() to check
  • Functions are objects with callable behavior
  • JavaScript variables can change types — dynamic typing means flexibility but requires caution
  • Use BigInt for numbers beyond 2^53 - 1
