JavaScript this Keyword

The this keyword refers to the object that is executing the current function. Its value depends on how the function is called.

👤 this in Object Method

const person = {
  firstName: "John",
  lastName: "Doe",
  fullName: function() {
    return this.firstName + " " + this.lastName;
  }
};
person.fullName(); // "John Doe"

📞 Explicit Binding with call()

const person1 = {
  fullName: function() {
    return this.firstName + " " + this.lastName;
  }
};
const person2 = {
  firstName: "John",
  lastName: "Doe"
};
person1.fullName.call(person2); // "John Doe"
Output will appear here.

🧠 Lesson Summary

In JavaScript, this refers to the execution context — the object that owns the current code. In global scope, this points to window (or undefined in strict mode). Inside object methods, this refers to the object itself. Arrow functions inherit this from their surrounding scope, making them ideal for callbacks. You can also manually set this using .call(), .apply(), or permanently bind it with .bind(). Understanding this is essential for writing predictable, context-aware code.

🧪 Try It Yourself

console.log(this); // → window (or undefined in strict mode)

  name: "CodeMaster",

    console.log(this.name); // → "CodeMaster"
  }
};
obj.greet();

const arrow = () => {
  console.log(this); // → inherits from outer scope
};
arrow();

function regular() {
  console.log(this); // → depends on how it's called

regular(); // → window or undefined
regular.call(obj); // → obj
