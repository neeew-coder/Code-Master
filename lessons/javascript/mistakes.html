JavaScript Common Mistakes

JavaScript is flexible, but that flexibility can lead to subtle bugs. This module highlights common mistakes and how to avoid them.

  • ❌ Using = instead of == or === in conditionals
  • ❌ Confusing addition with string concatenation
  • ❌ Breaking return statements across lines
  • ❌ Using named indexes in arrays
  • ❌ Misplacing semicolons or trailing commas
  • ❌ Comparing null before checking for undefined

📚 Syntax Reference

Safe Comparison:

if (x === 10) { ... } // strict comparison

Safe Return:

return a * 10; // never break return across lines

Safe Array Access:

const person = [];
person[0] = "John"; // ✅
person["firstName"] = "John"; // ❌ turns array into object

Safe Type Check:

if (typeof myObj !== "undefined" && myObj !== null) { ... }
