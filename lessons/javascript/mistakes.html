📚 Syntax Reference

Safe Comparison:

if (x === 10) { ... } // strict comparison

Safe Return:

return a * 10; // never break return across lines

Safe Array Access:

const person = [];
person[0] = "John"; // ✅
person["firstName"] = "John"; // ❌ turns array into object

Safe Type Check:

if (typeof myObj !== "undefined" && myObj !== null) { ... }

🧠 Lesson Summary

JavaScript’s flexibility makes it powerful — but also prone to subtle mistakes. Forgetting to declare variables, misusing this, ignoring strict mode, and writing deeply nested logic are common traps. Clean naming, modular structure, and proactive error handling help avoid these issues. Recognizing and correcting these patterns improves code quality, maintainability, and developer confidence.

🧪 Try It Yourself

// ❌ Mistake: undeclared variable
function greet() {
  name = "Mark"; // → global leak
  console.log(name);
}

// ✅ Fix
function greet() {
  const name = "Mark";
  console.log(name);
}

// ❌ Mistake: 'this' confusion
const obj = {
  name: "CodeMaster",
  greet: function() {
    setTimeout(function() {
      console.log(this.name); // → undefined
    }, 1000);
  }
};

// ✅ Fix with arrow function
const objFixed = {
  name: "CodeMaster",
  greet: function() {
    setTimeout(() => {
      console.log(this.name); // → "CodeMaster"
    }, 1000);
  }
};
