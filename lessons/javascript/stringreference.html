JavaScript String Reference

Strings in JavaScript are primitive and immutable. When you assign or modify strings, you're working with copies — not references. However, the String object behaves differently.

📄 Value vs Reference

// Primitive string
let a = "Hello";
let b = a;
b = "Hi";

console.log(a); // "Hello"
console.log(b); // "Hi"

// String object
let x = new String("Hello");
let y = x;
y.message = "Hi";

console.log(x.message); // "Hi"

Primitive strings are copied by value. new String() creates an object, which is copied by reference — allowing mutation.

🧠 Notes

  • Primitive strings are immutable and copied by value
  • new String() creates a wrapper object
  • String objects are copied by reference and can be mutated
  • Use primitives for most cases — they’re faster and safer
  • Use typeof to distinguish: "text" is string, new String() is object
