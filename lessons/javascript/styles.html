JavaScript Style Guide

Coding conventions improve readability, consistency, and maintainability. Follow consistent rules for naming, spacing, indentation, and structure across all your JavaScript projects.

  • Use camelCase for variables and functions
  • Use 2 spaces for indentation (no tabs)
  • Put spaces around operators and after commas
  • Use lowercase file names for consistency across servers
  • Break long lines after operators or commas
