JavaScript Number Reference

Numbers in JavaScript are primitive and immutable. When assigned to another variable, they’re copied by value. However, using new Number() creates a wrapper object, which behaves differently.

📄 Value vs Reference

// Primitive
let a = 100;
let b = a;
b = 200;

console.log(a); // 100
console.log(b); // 200

// Object wrapper
let x = new Number(100);
let y = x;
y.extra = "hello";

console.log(x.extra); // "hello"
console.log(x === y); // true

Primitive numbers are copied by value. new Number() creates an object, which is copied by reference — allowing mutation and shared identity.

🧠 Notes

  • Primitive numbers are immutable and copied by value
  • new Number() creates a wrapper object
  • Object numbers are copied by reference and can be mutated
  • typeof 100 is "number", but typeof new Number(100) is "object"
  • Use primitives for most cases — they’re faster and safer
