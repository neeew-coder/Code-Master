🐞 C# Debugging

Debugging is the process of identifying and resolving issues in your code. C# provides powerful tools like breakpoints, step execution, and logging to help you inspect and fix problems efficiently.

📘 What You'll Learn
  • Setting breakpoints and stepping through code
  • Using watch and locals windows
  • Logging with Console.WriteLine and Debug.WriteLine
  • Common debugging strategies

🔍 Explanation
  • Debug.WriteLine: Sends output to debug console
  • Breakpoints: Pause execution at specific lines
  • Step Into: Navigate method calls line-by-line
  • Watch Window: Monitor variable values during runtime

🧠 Analogy: Debugging is like detective work — you follow clues, inspect suspects (variables), and uncover the root cause of the problem.

📚 More Techniques

Conditional breakpoint:
Set breakpoint only when x > 10

Immediate window:
Evaluate expressions during paused execution

💡 Tips: Reproduce bugs consistently. Use logging for non-interactive debugging. Isolate logic to test in smaller units. Don’t ignore warnings.
