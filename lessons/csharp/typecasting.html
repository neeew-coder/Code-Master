🔄 C# Typecasting

Typecasting is the process of converting one data type into another. C# supports both implicit and explicit casting. Implicit casting happens automatically when there's no risk of data loss. Explicit casting requires manual conversion and may lose data.

📘 What You'll Learn
  • Difference between implicit and explicit casting
  • How to safely convert between types
  • Common pitfalls and how to avoid them
  • Using helper methods like Convert and Parse

🔍 Explanation
  • Implicit casting: Safe conversion (e.g., int to double)
  • Explicit casting: Manual conversion using (type)
  • Convert.ToX(): Converts between types with error handling
  • Parse(): Converts strings to numbers (throws error if invalid)

🧪 Analogy: Typecasting is like pouring water from a bottle into a glass. If the glass is big enough (implicit), it’s safe. If the glass is smaller (explicit), you might spill some — so you need to be careful and do it manually.

📚 More Examples

Explicit with potential data loss:
double value = 9.99;
int truncated = (int)value; // Result: 9

Safe string conversion:
string input = "123";
int number = Convert.ToInt32(input);

💡 Tips: Always validate user input before using Parse(). Prefer TryParse() when working with unpredictable data.
