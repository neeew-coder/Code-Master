⛔ C# Break & Continue

The break and continue statements give you control over loop execution. break exits the loop entirely, while continue skips the current iteration and moves to the next.

📘 What You'll Learn
  • How break exits a loop early
  • How continue skips to the next iteration
  • Common use cases for both
  • How to combine them with conditions

🔍 Explanation
  • break: Immediately exits the loop
  • continue: Skips current iteration and continues
  • Use if conditions to trigger them
  • Works in for, while, and foreach loops

🧠 Analogy: break is like leaving a meeting early. continue is like skipping one topic but staying for the rest.

📚 More Examples

Break on condition:
while (true) {
  if (exitRequested) break;
  Process();
}

Continue on filter:
foreach (var item in items) {
  if (!item.IsValid) continue;
  Process(item);
}

💡 Tips: Use break to exit loops when a goal is met. Use continue to skip unwanted data or conditions.
