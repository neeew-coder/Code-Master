🧪 C# Classes & Objects

Classes define the structure and behavior of objects. Objects are instances of classes, each with their own state and access to class-defined functionality.

🔍 Explanation
  • class Person: Defines a blueprint
  • new Person(...): Creates an object
  • Introduce(): Method tied to object behavior
  • Name and Age: Instance properties

🧠 Analogy: A class is like a cookie cutter. Each object is a cookie made from that mold, with its own flavor (state).

📚 More Examples

Default constructor:
public Person() {
  Name = "Unknown";
  Age = 0;
}

Object initialization:
var p = new Person { Name = "Alex", Age = 30 };

💡 Tips: Keep classes focused. Use constructors to enforce required data. Avoid exposing fields directly — use properties.

🧠 Lesson Summary

In C#, a class is a blueprint for creating objects — each object is an instance of a class with its own data and behavior. This tutorial shows how to define classes, create objects, and use them to model real-world concepts like users, products, or shapes. It’s the first step toward mastering object-oriented design.

🧪 Try It Yourself

// ClassObjectDemo.cs
using System;

class Car {
  public string brand;
  public int year;

  public void Honk() {
    Console.WriteLine("Beep beep!");
  }
}

class Program {
  static void Main() {
    Car myCar = new Car();
    myCar.brand = "Toyota";
    myCar.year = 2022;

    Console.WriteLine("Brand: " + myCar.brand);
    Console.WriteLine("Year: " + myCar.year);
    myCar.Honk();
  }
}
