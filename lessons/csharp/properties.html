🏡 C# Properties

Properties provide a flexible mechanism to read, write, or compute the values of private fields. They encapsulate data access while maintaining control and validation.

📘 What You'll Learn
  • Auto-implemented properties
  • Custom get and set accessors
  • Read-only and write-only properties
  • Property validation logic

🔍 Explanation
  • Auto-implemented: Simplified syntax for basic get/set
  • Custom accessors: Add logic to get or set
  • Validation: Prevent invalid assignments
  • Encapsulation: Protect internal fields while exposing safe access

🧠 Analogy: Properties are like smart windows — they let you see or change what's inside, but only under safe conditions.

📚 More Examples

Read-only property:
public int Id { get; } = 1001;

Write-only property:
private string secret;
public string Secret { set { secret = value; } }

💡 Tips: Use properties to enforce rules. Avoid exposing fields directly. Prefer auto-properties for simple data, and custom accessors for logic.
