🎯 C# Enums

Enums (short for “enumerations”) define a set of named constants. They improve code readability, enforce valid values, and support type-safe logic.

📚 More Examples

Default enum values:
enum Direction { North, East, South, West }
// North = 0, East = 1, ...

Custom starting value:
enum Priority { Low = 10, Medium = 20, High = 30 }

💡 Tips: Use enums for fixed categories. Avoid magic numbers. Prefer named values for clarity and maintainability.

🧠 Lesson Summary

Enums in C# are strongly typed constants that represent a fixed set of named values. This tutorial shows how to define enums, use them in control flow, and improve semantic clarity in your code. Enums are ideal for representing states, categories, roles, or options in a readable and maintainable way.

🧪 Try It Yourself

// EnumsDemo.cs
using System;

enum Day {
  Sunday,
  Monday,
  Tuesday,
  Wednesday,
  Thursday,
  Friday,
  Saturday
}

class Program {
  static void Main() {
    Day today = Day.Friday;

    switch (today) {
      case Day.Friday:
        Console.WriteLine("It's Friday! Time to celebrate.");
        break;
      default:
        Console.WriteLine("Just another day.");
        break;
    }
  }
}
