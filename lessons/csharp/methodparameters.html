🎯 C# Method Parameters

Parameters allow methods to receive input values. C# supports value parameters, reference parameters, optional parameters, and named arguments.

📄 Parameter Syntax

🔍 Explanation
  • ref: Passes by reference, must be initialized
  • out: Passes by reference, must be assigned inside method
  • Optional: Uses default value if not provided
  • Named: Allows calling with parameter names

🧠 Analogy: ref is like handing someone your notebook to edit. out is like giving them a blank page to fill. Optional parameters are like RSVP defaults.

📚 More Examples

Named arguments:
Greet(name: "Mark", timeOfDay: "afternoon");

Mixing optional and named:
Greet(timeOfDay: "night", name: "Mark");

💡 Tips: Use ref and out sparingly — they can reduce clarity. Prefer optional parameters for flexibility. Named arguments improve readability.

🧠 Lesson Summary

Method parameters in C# allow you to pass data into functions for processing. You can pass values directly, reference variables for modification, or use flexible argument lists. This tutorial explores how to define parameters, use them effectively, and build modular, reusable logic blocks for scalable applications.

🧪 Try It Yourself

// MethodParametersDemo.cs
using System;

class MethodParametersDemo {
  static void Main() {
    int result = Multiply(4, 5);
    Console.WriteLine("Product: " + result);

    Greet("Mark", 3);
  }

  static int Multiply(int a, int b) {
    return a * b;
  }

  static void Greet(string name, int times) {
    for (int i = 0; i < times; i++) {
      Console.WriteLine("Hello, " + name + "!");
    }
  }
}
