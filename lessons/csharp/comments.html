💬 C# Comments

Comments are notes in your code that are ignored by the compiler. They're used to explain logic, leave reminders, or document functionality. In C#, you can write single-line, multi-line, and XML-style documentation comments.

  • How comments improve code readability
  • How to use XML documentation comments
  • Best practices for commenting code

📄 Comment Syntax

Output will appear here.
🔍 Explanation
  • // starts a single-line comment.
  • /* ... */ wraps a multi-line comment.
  • /// is used for XML documentation comments.
  • Comments are ignored during execution.

🧠 Analogy: Think of comments like sticky notes on a recipe. They don’t change the dish, but they help you (or others) understand why certain steps are there. The compiler skips them — just like you’d skip reading the sticky notes while cooking.

📚 More Examples

Inline Comment:
int x = 5; // store initial value

Disabled Code:
// Console.WriteLine("This won't run");

💡 Tips: Use comments to explain why, not what. Avoid obvious comments like // increment x if the code is self-explanatory.
