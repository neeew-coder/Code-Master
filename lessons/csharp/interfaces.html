🧩 C# Interfaces

An interface defines a contract — a set of methods and properties that implementing classes must provide. It enables polymorphism, decouples logic, and supports multiple inheritance.

🧠 Analogy: An interface is like a job description — it tells you what must be done, but not how to do it.

📚 More Examples

Multiple interfaces:
interface IScanner { void Scan(); }
class MultiFunction : IPrinter, IScanner {
  public void Print() {}
  public void Scan() {}
}

Interface polymorphism:
IPrinter p = new MultiFunction();
p.Print(); // Calls MultiFunction.Print()

💡 Tips: Use interfaces to define roles. Prefer interface references for flexibility. Keep interfaces focused and cohesive.

🧠 Lesson Summary

Interfaces in C# define a set of methods and properties that implementing classes must provide. They allow you to write code that depends on behavior rather than concrete types, supporting polymorphism, loose coupling, and scalable architecture. This tutorial shows how to declare, implement, and use interfaces effectively.

🧪 Try It Yourself

// InterfaceDemo.cs
using System;

interface IAnimal {
  void Speak();
}

class Dog : IAnimal {
  public void Speak() {
    Console.WriteLine("Woof!");
  }
}

class Program {
  static void Main() {
    IAnimal pet = new Dog();
    pet.Speak(); // Output: Woof!
  }
}
